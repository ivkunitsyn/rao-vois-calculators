# Структура проекта RNK

## 1. Общая схема

Проект разделён на четыре независимые зоны:

- `calculators/` — расчётная логика.
- `interface/` — frontend-шелл и визуальный сценарий.
- `tools/` — служебные утилиты и бот обновления данных.
- `docs/` — документация по логике, эксплуатации и аудиту.

## 2. Калькуляторы

Каждый калькулятор имеет отдельную папку и собственный Python entrypoint `backend/calculator.py`.

### 2.1. РАО — вещатели телеканалов

- Папка: `calculators/rao_tv/`
- Backend API: `calculators/rao_tv/backend/api.py`
- Расчётный движок: `calculators/rao_tv/backend/rao.py`
- Python entrypoint калькулятора: `calculators/rao_tv/backend/calculator.py`
- Данные:
  - `calculators/rao_tv/data/Таблица РКН.xlsx`
  - `calculators/rao_tv/data/Таблица РКН slim.xlsx`
  - `calculators/rao_tv/data/Таблица РКН.sqlite`
  - `calculators/rao_tv/data/Переменные из ставок.xlsx`
  - `calculators/rao_tv/data/inn_name.csv`

### 2.2. РАО — вещатели радиоканалов

- Папка: `calculators/rao_radio/`
- Python entrypoint: `calculators/rao_radio/backend/calculator.py`
- Статус: логика ещё не реализована, модуль оставлен отдельно специально, чтобы радиологика не смешивалась с ТВ.

### 2.3. РАО — трансляторы мероприятий

- Папка: `calculators/rao_events/`
- Python entrypoint и логика: `calculators/rao_events/backend/calculator.py`

### 2.4. ВОИС — вещатели телеканалов

- Папка: `calculators/vois_tv/`
- Python entrypoint: `calculators/vois_tv/backend/calculator.py`
- Таблица параметров: `calculators/vois_tv/data/Переменные из ставок ВОИС.xlsx`
- Сейчас использует общий ТВ-движок `calculators/rao_tv/backend/rao.py`, но вызывается в отдельном режиме `calc_type=vois_tv`.

### 2.5. ВОИС — вещатели радиоканалов

- Папка: `calculators/vois_radio/`
- Python entrypoint: `calculators/vois_radio/backend/calculator.py`
- Статус: отдельный placeholder, без смешения с логикой ТВ.

### 2.6. ВОИС — трансляторы мероприятий

- Папка: `calculators/vois_events/`
- Python entrypoint и логика: `calculators/vois_events/backend/calculator.py`

## 3. Frontend

- Общий frontend находится в `interface/index.html`.
- Внутри него есть меню выбора калькулятора и отдельные ветки состояния:
  - `rao_tv`
  - `vois_tv`
  - `rao_events`
  - `vois_events`
  - `rao_radio` и `vois_radio` пока выведены как placeholder.

Ключевой момент:

- backend-логика калькуляторов разделена;
- frontend пока общий, но маршрутизирует пользователя по отдельным калькуляторам и хранит состояние отдельно по каждому калькулятору.

## 4. Цепочка данных РКН

### 4.1. Источник

Сотрудники загружают сырую таблицу РКН через Telegram-бот:

- `tools/rkn_table_bot/rkn_telegram_bot.py`

### 4.2. Очистка

Очистка выполняется модулем:

- `tools/rkn_table_bot/rkn_cleaner.py`

Шаги очистки:

1. Оставляет только поля, нужные для расчёта.
2. Исключает строки с недействующим статусом лицензии.
3. Исключает строки без ИНН и без номера лицензии.
4. Нормализует численность населения.
5. Строит артефакты для быстрого поиска:
   - очищенный XLSX;
   - `inn_name.csv`;
   - SQLite-индекс.

### 4.3. Рабочие файлы

По умолчанию бот обновляет именно рабочие данные проекта:

- `calculators/rao_tv/data/Таблица РКН slim.xlsx`
- `calculators/rao_tv/data/Таблица РКН.sqlite`
- `calculators/rao_tv/data/inn_name.csv`

Это важно, потому что API ищет данные в первую очередь именно там.

### 4.4. Подхват приложением

`calculators/rao_tv/backend/api.py` умеет:

- брать `inn_name.csv` для быстрого имени организации;
- брать SQLite-индекс для быстрого поиска лицензий;
- при отсутствии SQLite откатываться к XLSX;
- пересобирать индекс в памяти по изменению `mtime` файла.

Значит, новая таблица, загруженная через бот, не требует ручной правки кода: backend подхватит обновлённые артефакты автоматически.

## 5. Git и сервер

Бот может не только обновлять рабочие файлы на сервере, но и при включённых env-параметрах:

- зафиксировать обновлённые артефакты в git;
- при необходимости отправить их в удалённый репозиторий.

Это делается только для явного набора файлов данных и не затрагивает остальные изменения в рабочем дереве.

## 6. Что уже хорошо разделено

- события РАО и ВОИС — отдельные Python-модули;
- радио-калькуляторы не смешаны с ТВ;
- бот обновления таблицы и очистка вынесены в отдельный `tools/`;
- данные РКН отделены от логики расчёта;
- frontend физически отделён от backend.

## 7. Что ещё остаётся общей частью

- TV API сейчас единый (`calculators/rao_tv/backend/api.py`) для РАО ТВ, ВОИС ТВ и калькуляторов мероприятий.

Это не ломает работу, но это главный общий слой, который при дальнейшем рефакторинге логично будет дробить на отдельные router-модули по калькуляторам, если понадобится ещё более жёсткая изоляция.
