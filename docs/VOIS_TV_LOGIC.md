# ВОИС ТВ: схема логики расчёта

Источник: `Polozhenie-3-o-stavkah-VOIS-TV.pdf` (редакция от 26.12.2024).

## 1) Входные данные

- ИНН и наименование организации.
- База для процента: доходы (квартал / год÷4) или, при спец-случае, расходы.
- Набор выбранных вещательных лицензий.
- Среда договора: `auto` / `в эфире или по кабелю` / `одновременно в эфире и по кабелю`.
- Население по лицензиям (авто из РКН + ручное подтверждение/корректировка).
- Количество интернет-ресурсов (сайты/приложения).
- Признак ассоциации, нового пользователя, квартал договора.
- Опционально: количество абонентов, сумма выплат по проценту за прошлый год (для ветки гильотины).

## 2) Процентная ставка (раздел 2 + Приложение №2)

1. Для каждой лицензии считается ставка телеканала по тематикам:
   - преобладающая тематика >50% -> ставка категории этой тематики;
   - иначе средневзвешенная по долям тематик;
   - если долей нет -> среднее по перечисленным тематикам (равные доли);
   - если тематика не отнесена к категориям -> 1,5%.
2. Если в лицензии несколько СМИ/каналов — внутри лицензии ставка усредняется по часам вещания.
3. По договору ставка усредняется по всем лицензиям с весом `часы × население`.
4. Округление: до 0,1%.

Категории ставок ВОИС:
- I: 1,0%
- II: 1,3%
- III: 1,5%
- IV: 1,7%
- V: 2,0%

## 3) Сумма по проценту (раздел 2)

`S_percent_q = round(base_q * contract_rate / 100)`

Где `base_q` — квартальная база (доходы за квартал либо год/4, либо расходы).

## 4) Минимальная сумма (раздел 3)

### 4.1 Базовая минималка по лицензиям (п.3.1)

Для каждой лицензии выбирается строка таблицы по диапазону населения и среде вещания:
- `В эфире или по кабелю`
- `Одновременно в эфире и по кабелю`

Далее суммирование по лицензиям.

### 4.2 Ветка малого дохода (п.3.2)

Если годовой доход <= 1 500 000 и период допустим:
- минималка = `0,5 ×` минималка по суммарному населению,
- среда: как для договора,
- действует ограниченный срок (до 8 полных периодов).

### 4.3 Снижение по числу лицензий (п.3.3)

После суммирования применяется коэффициент по числу лицензий:
- 4–6: 5%
- 7–9: 9%
- 10–12: 16%
- 13+: 28%

### 4.4 Альтернативы по территории/абонентам (п.3.4–3.5)

- При подтверждённых данных о фактической территории — можно считать по фактическому населению.
- При режиме абонентов: минимум `5 ₽ × абоненты`.

### 4.5 Интернет-доплата (п.3.6)

Для пользователей 1.2.2:
- увеличение на 15% от минималки,
- но не менее 12 500 ₽ за каждый ресурс.

## 5) Гильотина (п.3.7)

Если минималка > `1,10 × S_percent_q`, применяется последовательность:
1. Минималка по п.3.1 от суммарного населения (независимо от числа лицензий).
2. Для ветки п.3.2: если всё ещё > 1,10 × процента, то `1/4` от фактических выплат по проценту за предшествующие 4 квартала (или fallback на текущую процентную сумму при отсутствии данных).

## 6) Коэффициенты (раздел 4)

- По часам вещания (п.4.1): 0,8 / 0,6 / 0,4 при <126 часов.
- По категориям пользователей (п.4.2): применяется только один коэффициент из доступных.
- Новый пользователь по периодам (п.4.3): 0,75 (1–4 кв), 0,88 (5–8 кв), только к минималке.

## 7) Финал

`S_pay = max(S_percent_q, S_min_q)`

Отчёт выводит:
- ставку договора,
- сумму по проценту,
- минимальную сумму,
- итог к оплате,
- примечания и применённые коэффициенты.
